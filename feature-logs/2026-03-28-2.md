## Summary

1. **LSP Diagnostics Pipeline** - Added end-to-end diagnostics flow from `publishDiagnostics` to squiggles, gutter markers, hover messages, and a Problems panel.
2. **Structural Code Navigation** - Added go-to-definition, find references, rename symbol, and symbol-result navigation UI.

## 1. LSP Diagnostics Pipeline

### Overview

Implemented the missing diagnostics path so language servers can push errors and warnings into the editor instead of only exposing completion and hover. Diagnostics now arrive from the Rust transport layer, are stored in frontend state, rendered inline in CodeMirror, and surfaced in a Problems panel.

| `src/hooks/useLspDiagnostics.ts` | Frontend event subscription and diagnostics hydration flow |
| `src/components/problems/ProblemsPanel.tsx` | Problems list UI with file grouping and click-to-jump behavior |
| `src/stores/diagnosticsStore.ts` | Shared diagnostics state keyed by normalized file path |

### Files Modified

|------|---------|
| `src-tauri/src/lsp/transport.rs` | Routed `publishDiagnostics` notifications into the app event flow |

| `src-tauri/src/commands/lsp_commands.rs` | Added diagnostics listing surface for the frontend |

| `src/index.css` | Added squiggle, gutter, hover, and jump-flash styling |

### Implementation Details

#### Diagnostics Flow

```
┌────────────────────────┐     ┌──────────────────────┐     ┌─────────────────────────┐
│ Language server emits  │────►│ Rust LSP transport   │────►│ frontend diagnostics     │
│ publishDiagnostics     │     │ / manager            │     │ store                    │
└────────────────────────┘     └──────────────────────┘     └────────────┬────────────┘
                                                                         │
                                                                         ▼
                                                        ┌─────────────────────────────────┐
                                                        │ CodeEditor squiggles / gutter / │
                                                        │ Problems panel / status badge   │
                                                        └─────────────────────────────────┘
```

#### Rendering Changes

- removed the heavy background-fill error styling
- switched to squiggle-first underlines
- added per-line gutter dots
- added hover messages for diagnostics
- styled severity levels separately for error, warning, info, and hint

### Architecture Decisions

1. **Normalize paths at store boundaries** - LSP diagnostics, file tabs, and Problems navigation must all agree on the same path key shape.
2. **Store first, editor second** - Problems, status counts, and editor decorations all read from one diagnostics source of truth.
3. **Use lightweight CodeMirror decorations** - Kept the editor native instead of adding a second diagnostics abstraction layer.

## 2. Structural Code Navigation

### Overview

Extended the LSP client beyond completion/hover and added real IDE-style symbol navigation. The editor can now request definitions, references, and rename operations from the active language server, then open results in reusable UI rather than forcing manual file search.

### Files Created

| File | Purpose |
|------|---------|
| `src/components/navigation/SymbolsPanel.tsx` | Sidebar UI for definitions/references result sets |
| `src/stores/navigationStore.ts` | Shared state for definition/reference result groups and jump targets |

### Files Modified

| File | Changes |
|------|---------|
| `src-tauri/src/lsp/protocol.rs` | Added request builders for definition, references, and rename |
| `src-tauri/src/commands/lsp_commands.rs` | Added Tauri commands for definition, references, and rename |
| `src/components/editor/CodeEditor.tsx` | Added keyboard bindings, request triggers, and result handling for structural navigation |
| `src/components/layout/Sidebar.tsx` | Added symbol-results sidebar rendering |

### Implementation Details

#### Navigation Flow

```typescript
await invoke("lsp_definition", { path, line, character });
await invoke("lsp_references", { path, line, character });
await invoke("lsp_rename", { path, line, character, newName });
```

The editor now supports:
- `F12` for definition
- `Shift+F12` for references
- `F2` for rename

Single-result navigations jump directly. Multi-result references are grouped in the Symbols panel for preview and selection.

### Architecture Decisions

1. **Direct jump for one result, panel for many** - Keeps simple navigations fast while handling references in a usable way.
2. **Reuse sidebar real estate** - Symbol results fit naturally beside Explorer/Search/Problems rather than opening floating modals.

## 3. Diagnostics UX and Navigation Fixes

### Overview

Several follow-up bugs were fixed while hardening the new LSP surfaces. These were required to make diagnostics and Problems navigation feel stable enough for daily use.

### Files Created

| File | Purpose |
|------|---------|
| `src/utils/path.ts` | Shared path normalization helpers for LSP/editor/file state interop |

### Files Modified

| File | Changes |
|------|---------|
| `src/hooks/useFileSystem.ts` | Normalized navigation/open-file paths to avoid duplicate tabs |
| `src/stores/fileStore.ts` | Aligned tab/file lookup with normalized paths |
| `src/components/editor/CodeEditor.tsx` | Added temporary line flash when jumping from Problems |
| `src/components/layout/Sidebar.tsx` | Added Problems error badge integration |
| `src/components/layout/SidebarNav.tsx` | Fixed Zustand selector loop and rendered Problems alert dot safely |
| `src-tauri/src/commands/file_watcher.rs` | Fixed watcher panic that could tear down editor state |

### Gotchas and Fixes

1. **Duplicate empty tabs from Problems clicks** - Caused by mismatched path formats between LSP URIs and editor-opened files.
2. **CodeMirror gutter crash** - Gutter markers must be added in sorted order; unsorted diagnostic ranges caused a runtime exception.
3. **Infinite render loop in `SidebarNav`** - A fresh Zustand object selector caused `getSnapshot` churn and a max update depth failure.

### Architecture Decisions

1. **Normalize once, everywhere** - Prevented subtle Windows path mismatches from resurfacing in multiple features.
2. **Flash target line on jump** - Gives clear spatial feedback when navigating from Problems into dense files.

| `src/components/settings/SettingsPage.tsx` | Added LSP Extensions settings page and actions |

### Implementation Details

#### Managed Runtime Model

```
┌────────────────────┐
│ lsp-catalog.json   │
└─────────┬──────────┘
          │
          ▼
┌────────────────────┐     install/update     ┌─────────────────────────┐
│ lsp_runtime.rs     │───────────────────────►│ app data / lsp-runtimes │
│ status + commands  │                        │ metadata + executables  │
└─────────┬──────────┘                        └────────────┬────────────┘
          │                                                │
          └──────────── resolve command at runtime ────────┘
```

#### Node Runtime Change

TypeScript and Pyright were initially pinned to stale versions. The install path was updated to install `latest` and then read the actual installed version from `node_modules/<pkg>/package.json` so the UI still shows a real version.

### Architecture Decisions

1. **Bundled catalog first** - Enables controlled app-managed installs without taking on remote infrastructure immediately.
2. **Keep install provider selectable** - `pnpm` stays the default, but `npm` remains available for environments where `pnpm` is absent.
3. **Managed-first source visibility** - The UI now shows whether a runtime is coming from `managed`, `path`, or is still `missing`.

## 5. Startup Auto-Install + Activity Bar Settings

### Overview

Added a startup bootstrap step that ensures the built-in TypeScript, Python, and Rust runtimes install automatically when the app starts. Also added a switchable VS Code-style activity bar so sidebar icons can move into a dedicated left rail with top/bottom alignment settings.

### Files Created

| File | Purpose |
|------|---------|
| `src/components/layout/SidebarNav.tsx` | Reusable sidebar/activity bar control strip |

### Files Modified

| File | Changes |
|------|---------|
| `src/App.tsx` | Bootstraps the LSP extensions hook at startup |
| `src/hooks/useLspExtensions.ts` | Added one-time default-runtime ensure flow on app startup |
| `src/components/layout/MainLayout.tsx` | Added activity bar rendering mode |
| `src/components/layout/Sidebar.tsx` | Integrated activity bar / integrated-strip switching |
| `src/components/settings/SettingsPage.tsx` | Added install-provider selector and activity bar settings |

### Implementation Details

#### Startup Ensure Flow

```
┌──────────────┐     ┌─────────────────────┐     ┌────────────────────────────┐
│ App mount    │────►│ useLspExtensions()  │────►│ ensure bundled runtimes    │
└──────────────┘     └──────────┬──────────┘     │ TS / Py / Rust are managed │
                                │                └────────────────────────────┘
                                ▼
                    ┌────────────────────────┐
                    │ Extensions UI shows    │
                    │ source + install state │
                    └────────────────────────┘
```

### Architecture Decisions

1. **Auto-install on startup, not prelaunch block** - Keeps the app responsive while still converging toward managed runtimes automatically.
2. **Session-scoped bootstrap guard** - Prevents duplicate auto-installs when both `App` and Settings mount extension hooks.
3. **Separate activity bar component** - Avoids mixing navigation logic into Explorer/Search/Problems content containers.

## Testing Instructions

### Diagnostics

1. Open a `.ts`, `.py`, or `.rs` file with an obvious type error.
2. Hover the squiggle and open the Problems panel.
3. **Expected**: diagnostic message appears inline, the Problems icon shows an error dot, and clicking the problem jumps to the correct line with a flash highlight.

### Code Navigation

1. Open a symbol with usages in a supported file.
2. Press `F12`, `Shift+F12`, and `F2`.
3. **Expected**: definition jumps directly, references open in the Symbols panel, and rename sends the LSP request.

3. **Expected**: TypeScript, Python, and Rust begin installing automatically on startup and the card source eventually shows `managed`.

1. Open `Settings` → `Appearance`.
2. Switch navigation mode to `activity_bar`.
3. Change alignment between `top` and `bottom`.
4. **Expected**: Explorer/Search/Problems/Symbols icons move to a dedicated left rail and respect the chosen alignment.
