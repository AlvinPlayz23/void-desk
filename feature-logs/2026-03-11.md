## 1. Conversation Context / Memory Fix

### Overview

The app UI was persisting chat messages per session, but the SDK request pipeline was relying only on backend in-memory session state when building model history. That meant the model could receive a fresh conversation even while the UI still showed prior turns. The fix makes each request resend the active session's prior chat history so the model keeps context within the configured window.

### Files Modified

| File | Changes |
|------|---------|
| `src/hooks/useAI.ts` | Serializes prior session messages and sends them with each `ask_ai_stream_with_session` request. |
| `src-tauri/src/commands/ai_commands.rs` | Accepts `historyMessages`, resolves effective request history, trims it to a context window, and uses it in the SDK request pipeline. |
| `src-tauri/src/sdk/provider/mod.rs` | Added simple model context-window inference helper. |
| `src-tauri/src/sdk/provider/openai_compatible.rs` | Exposes inferred model context window in `ModelInfo`. |

### Implementation Details

#### Root Cause

The visible session transcript lived on the frontend, but `process_ai_stream()` could rebuild context from only the Rust in-memory `SessionStore`. If backend state was missing or reset, the request history sent to the model lost prior turns.

#### Fix Strategy

1. Serialize current session's prior `user`/`assistant` messages in the frontend.
2. Send that history with every session-based AI request.
3. Use provided history as the request context in the backend.
4. Trim older messages when the configured context window is exceeded.
5. Keep history scoped to the active session only.

### Architecture Decisions

1. **Use real session transcript as request source** - More reliable than depending only on ephemeral backend memory.
2. **Keep implementation simple** - No summarization, retrieval, replay, or compression.
3. **Drop old context naturally** - Once the budget is exceeded, oldest turns fall out of context.

## 2. Configurable Context Window

### Overview

Added a `Conversation Context Window` setting in AI Settings so the app can control how much prior chat history gets resent to the model. This is a simple approximate token budget, not exact tokenizer accounting.

### Files Modified

| File | Changes |
|------|---------|
| `src/stores/settingsStore.ts` | Added persisted `chatContextWindow` setting and setter. |
| `src/components/settings/SettingsPage.tsx` | Added numeric AI Settings control for context window tokens. |

### Implementation Details

- Default context window: `32000`
- Minimum enforced: `1024`
- Older turns are trimmed first
- If known model capacity is lower than requested, the request uses the lower value
